Module 1

Connecter vos agents à Claire

Claire reçoit les événements de vos automatisations via un webhook HTTPS simple. Aucun SDK à installer, aucune modification de code existant.

🔮 Make (Integromat)
Ajoutez un module HTTP dans votre scénario pour envoyer les événements à Claire.
Webhook
⚡ n8n
Utilisez le nœud HTTP Request pour envoyer les logs à l'endpoint Claire.
Webhook
🧠 Claude (Anthropic)
Interceptez les réponses de l'API Claude et retransmettez-les à Claire.
API hook
🤖 OpenAI
Middleware compatible GPT-3.5/4/o. Logging transparent sans latence ajoutée.
Middleware
🔮
Connexion Make — pas à pas
5 étapes · ~15 minutes
1
Dans votre scénario Make, ajoutez un module HTTP → Make a request à la fin de votre workflow.
2
Configurez l'URL : https://votre-instance-claire.io/api/events — méthode : POST
3
Dans le body JSON, mappez les champs de votre scénario :
// Body JSON à envoyer depuis Make
{
  "agent_id":   "{{votre_scenario_id}}",
  "agent_name": "{{nom_du_scénario}}",
  "status":     "{{1.status}}",          // running | error | success
  "message":    "{{1.result}}",          // résultat ou message d'erreur
  "timestamp":  "{{formatDate(now; iso8601)}}",
  "integration":"make"
}
4
Ajoutez le header d'authentification : Authorization: Bearer VOTRE_CLE_API_CLAIRE
5
Lancez un test de votre scénario. L'agent doit apparaître dans votre tableau de bord Claire sous 60 secondes.
⚡
Connexion n8n — pas à pas
4 étapes · ~10 minutes
1
Dans votre workflow n8n, ajoutez un nœud HTTP Request à la fin de votre chaîne.
2
Méthode : POST · URL : https://votre-instance-claire.io/api/events
3
Authentication : Header Auth · Name : Authorization · Value : Bearer VOTRE_CLE
// Body JSON (JSON/RAW dans n8n)
{
  "agent_id":    "{{ $workflow.id }}",
  "agent_name":  "{{ $workflow.name }}",
  "status":      "{{ $runIndex > 0 ? 'success' : 'running' }}",
  "message":     "{{ JSON.stringify($json) }}",
  "timestamp":   "{{ new Date().toISOString() }}",
  "integration": "n8n"
}
4
Activez et exécutez votre workflow. Vérifiez l'apparition de l'agent dans Claire.

Module 2

Comprendre le tableau de bord

Le dashboard centralise l'état de tous vos agents. Voici comment lire chaque indicateur et configurer les alertes adaptées à votre activité.

📊
Lecture des KPIs
Ce que chaque chiffre signifie pour votre activité
Indicateur Signification Seuil d'alerte recommandé
Score de santé Synthèse pondérée : taux d'erreur, latence, anomalies détectées. 100 = parfait. Alerte si < 70
Latence (ms) Temps moyen de réponse de l'agent sur les 15 dernières minutes. Alerte si > 2 000 ms
Événements / h Nombre d'exécutions de l'agent sur la dernière heure. Alerte si chute > 50%
Anomalies Comportements hors-norme détectés (timeout, erreur répétée, déviation de prompt). Alerte si > 3 en 1h
Statut Running · Error · Paused · Idle Alerte immédiate si Error
🔔
Configurer les alertes
Notifications email et Slack
1
Dans Claire → Paramètres → Alertes, cliquez sur "Nouvelle alerte".
2
Sélectionnez le trigger : Score de santé < seuil · Statut = Error · Anomalies > N en X min
3
Configurez le canal : email (multi-destinataire) ou Slack webhook. Testez avant de sauvegarder.
4
Recommandation : créez au moins 2 alertes — une critique (Error) et une préventive (Score < 70).

Module 3

Agir sur un agent — sans code

L'onglet Contrôle vous donne les commandes directes sur chaque agent. L'onglet Diagnostic vous aide à comprendre et résoudre les incidents.

⚡
Onglet Contrôle — actions directes
▶
Pause / Reprendre — Stoppe l'exécution de l'agent. Les événements en file d'attente sont conservés et relancés automatiquement à la reprise.
⧖
Throttle — Réduit le débit de l'agent (% d'appels autorisés/min). Utile lors de pics de charge ou de quotas API approchants.
↩
Rollback — Revient à la configuration précédente de l'agent (prompt, paramètres). Utilisez si un changement récent a dégradé les performances.
🔍
Onglet Diagnostic — Root Cause Analysis
1
Sélectionnez l'agent en anomalie dans la liste. Son statut sera Error ou son score de santé bas.
2
Cliquez sur "Lancer l'analyse RCA". Claire analyse les derniers logs, patterns et corrélations temporelles (3–5 secondes).
3
Le résultat affiche : cause racine identifiée · impact estimé · timeline de l'incident.
4
Les Playbooks de remédiation apparaissent automatiquement sous la RCA. Cliquez sur "Appliquer" pour chaque étape validée.

Module 4

Gouvernance & Sécurité IA

Claire vous permet de surveiller la fiabilité comportementale de vos agents — pas seulement leur disponibilité — et de sécuriser vos données en transit.

🛡
Guardrails IA — onglet Contrôle
Protections actives configurables sans code
Détection PII (données personnelles)
Bloque automatiquement toute donnée personnelle (nom, email, IBAN, numéro de téléphone) avant qu'elle ne soit transmise au LLM. Essentiel pour la conformité RGPD.
Prévention Prompt Injection
Détecte les tentatives d'injection dans les inputs utilisateurs (ex : "Ignore les instructions précédentes et…"). Protège l'intégrité de vos automatisations.
Faithfulness (cohérence des réponses)
Compare les réponses de l'agent avec les sources de référence. Signale les hallucinations ou réponses non fondées. Recommandé pour les agents de conseil ou SAV.
Filtre contenu non-conforme
Bloque les réponses contenant du contenu inapproprié, hors politique d'usage, ou contraire à la charte de votre entreprise.
📈
Score AgentOps — onglet Gouvernance
4 métriques de fiabilité comportementale
MétriqueCe qu'elle mesureSeuil de vigilance
Instruction Adherence L'agent suit-il les instructions que vous lui avez données ? Score 0–100. Alerte si < 75
Tool Accuracy L'agent utilise-t-il les bons outils (APIs, fonctions) dans le bon ordre ? Alerte si < 70
Context Relevance Les réponses sont-elles pertinentes par rapport au contexte fourni ? Alerte si < 70
Answer Correctness Les réponses sont-elles factuellement correctes par rapport aux sources ? Alerte si < 70

Support

Questions fréquentes

Mon agent n'apparaît pas dans le dashboard après connexion. Que faire ?
Vérifiez d'abord que votre webhook envoie bien un code HTTP 200 depuis Make/n8n (l'historique d'exécution le montre). Ensuite, vérifiez que le champ agent_id est unique et non-vide. Si le problème persiste après 2 min, contactez le support avec le contenu exact du body envoyé.
Comment modifier les seuils d'alerte après la configuration initiale ?
Dans Claire → Paramètres → Alertes, cliquez sur l'alerte existante et modifiez le seuil. Les changements sont appliqués en temps réel, sans redémarrage.
Les guardrails impactent-ils la latence de mes agents ?
La détection PII et l'injection ajoutent environ 8–15ms de latence (traitement côté Claire, pas côté LLM). La vérification Faithfulness est asynchrone et n'impacte pas le temps de réponse de votre agent.
Puis-je connecter plusieurs scénarios Make au même agent dans Claire ?
Oui. Utilisez le même agent_id dans tous vos scénarios. Claire agrège automatiquement les événements par agent. Chaque scénario peut être identifié via le champ source optionnel dans le body.
Mes données sont-elles stockées par Claire ?
Les payloads de logs sont chiffrés au repos (AES-256) et en transit (TLS 1.3). Les données sont stockées dans votre instance dédiée. Aucun partage tiers. Conformité RGPD assurée. Sur demande : DPA disponible.
